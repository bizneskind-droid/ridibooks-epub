# ridibooks-epub

Скачивает главы книги с [ridibooks.com](https://ridibooks.com) и собирает их в один файл EPUB.

## Возможности

- Проходит защиту Cloudflare через [cloakbrowser](https://cloakbrowser.dev) (стелс-хромиум) — обычный Playwright/agent-browser блокируется.
- Тянет название книги и глав из `og:title`, обложку — с CDN ridibooks.
- Собирает валидный EPUB3 (оглавление, обложка, CSS) целиком в памяти, без промежуточных файлов на диске.
- Упавшие главы пропускаются, а не роняют весь прогон.

## Установка

С помощью [uv](https://github.com/astral-sh/uv) (быстрее):

```bash
uv sync
```

`uv sync` читает `pyproject.toml`, создаёт `.venv` и ставит зависимости из
`uv.lock`. Если предпочитаешь `requirements.txt`:

```bash
uv venv
uv pip install -r requirements.txt
```

Либо через обычный venv + pip:

```bash
python3 -m venv venv
./venv/bin/pip install -r requirements.txt
```

Бинарник cloakbrowser (~200 МБ) скачается автоматически при первом запуске.

## Использование

```bash
uv run ridibooks.py      # если ставили через uv
./venv/bin/python ridibooks.py   # если через pip
```

Скрипт спросит адреса первой и последней главы:

```
Введите адрес первой главы: https://ridibooks.com/books/425270925/view
Введите адрес последней главы: https://ridibooks.com/books/425270949/view
```

На выходе — файл `<название книги>.epub`.

## Докачка (кэш)

Каждая скачанная глава кладётся в `cache/<book_id>.json` рядом со скриптом.
При повторном запуске уже скачанные главы берутся из кэша мгновенно — качаются
только недостающие (например, если глава в прошлый раз упала по таймауту). Если
все главы в кэше, браузер (Chromium) вообще не запускается.

Чтобы перекачать заново — удали папку `cache/` (или отдельный `cache/<id>.json`).

## Платные главы: как вложить куки

Без куки качаются только бесплатные превью-главы. Чтобы скачать купленные главы,
нужно передать скрипту свою залогиненную сессию ridibooks. Самый простой способ —
экспортировать куки расширением браузера:

1. Залогинься на **ridibooks.com** в обычном браузере.
2. Поставь расширение **«Get cookies.txt LOCALLY»**
   ([Chrome](https://chromewebstore.google.com/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc),
   Firefox — то же по названию). Оно работает локально, ничего никуда не шлёт.
3. Открой ridibooks.com, нажми на иконку расширения → **Export** → сохрани файл.
4. Положи файл `cookies.txt` **рядом с `ridibooks.py`** и запусти скрипт как обычно.

Скрипт сам подхватит `cookies.txt` и напишет `Куки загружены (N шт.)`.

Альтернативы (то же самое, разными способами):

| Способ | Файл | Куда класть |
|---|---|---|
| Расширение «Get cookies.txt LOCALLY» | `cookies.txt` | рядом со скриптом |
| Playwright `storage_state` (куки + localStorage) | `storage_state.json` | рядом со скриптом |
| Свой путь к любому из файлов | — | `RIDI_COOKIES=/путь/к/файлу uv run ridibooks.py` |

Куки живут недолго (ridibooks их периодически инвалидирует) — если платные главы
снова стали превью, экспортируй `cookies.txt` заново.

> ⚠️ `cookies.txt` = доступ к твоему аккаунту. Не коммить его и не передавай никому
> (он уже в `.gitignore`).

## Как это устроено

| Данные | Источник |
|---|---|
| Текст главы | `#viewer_contents` (стабильный `id`, ждём загрузки JS-читалки) |
| Название книги + номер главы | `<meta property="og:title">` |
| Обложка | `https://img.ridicdn.net/cover/{book_id}/xxlarge` (480×689; без `/xxlarge` отдаёт 120×172) |

Селекторы вынесены в константы вверху `ridibooks.py` — если фронтенд ridibooks изменится, править там.

## Заметки / подводные камни

- **ID глав идут не строго +1.** Скрипт перебирает диапазон `id` и пропускает несуществующие. Если между первой и последней главой много «дыр», часть запросов уйдёт впустую.
- **`inner_html()` отдаёт HTML, а не XHTML** (незакрытые `<br>` и т.п.). Функция `html_to_xhtml()` пересобирает фрагмент через lxml, иначе ebooklib молча вернёт пустое тело и упадёт на генерации навигации.
- **Без пролога `<?xml ... encoding?>`** в контенте главы: lxml не парсит `str` с объявлением encoding — ebooklib тоже отдаст пустое тело. Свой пролог ebooklib дописывает сам при записи.
- **VPS (Ubuntu 24.04+, AppArmor):** нужен `--no-sandbox` — уже прописан в `launch()`.
- **Платный контент:** первые главы обычно открыты как превью. Для платных глав положи `cookies.txt` рядом со скриптом (см. раздел «Платные главы: как вложить куки»).
